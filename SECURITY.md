# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.2.x   | ✅        |
| < 0.2   | ❌        |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report security issues privately via one of these channels:

- **GitHub Private Advisory**: [Report a vulnerability](https://github.com/llmhut/agentvm/security/advisories/new) ← preferred
- **Email**: security@llmhut.dev (if set up)

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### What to expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- **Fix timeline** communicated as soon as scope is understood
- **Credit** in the release notes if you'd like it

We follow [Responsible Disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure) — please give us reasonable time to patch before going public.
